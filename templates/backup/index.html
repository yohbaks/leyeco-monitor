Backup Best Practices
  • Download a full backup at least weekly
  • Store backups in a separate location (USB drive, Google Drive, email)
  • Run a backup before any system updates
  • Test opening the Excel files to verify they're readable
  • Label backup files with the date (auto-included in filename)
Scheduled Backup (Command Line)

Run this command to create a backup from the command line (useful for Windows Task Scheduler or cron):

python manage.py backup_now

Saves the backup ZIP to the backups/ folder in the project directory.

Important Note
Backup files contain sensitive financial and personal data. Do not share them with unauthorized persons. The download is logged in the audit trail.
